# clipImg Rust 单 EXE 方案实施计划

## Context

当前 clipImg 由 PowerShell 守护进程 + AutoHotkey 热键脚本两部分组成，需要分别启动、分别维护。目标是合并为一个 Rust 单 EXE，支持可配置热键和路径，体积控制在 5MB 以内。

## 技术选型

| 组件 | Crate | 说明 |
|------|-------|------|
| 剪贴板图片读取 | `arboard` 3.x | 1Password 维护，直接输出 RGBA 像素数据，无需手写 Win32 FFI |
| 图片编码保存 | `image` 0.25 | RGBA → PNG 编码 |
| 全局热键 | `global-hotkey` 0.6 | Tauri 出品，支持字符串解析热键（`"Alt+Insert"`），运行时可重注册 |
| 系统托盘 | `tray-icon` 0.21 + `muda` 0.17 | Tauri 出品，右键菜单+tooltip |
| 事件循环 | `tao` 0.3 | Tauri 对 winit 的 fork，与 global-hotkey/tray-icon 深度集成 |
| 键盘模拟 | `windows-sys` | 直接调用 Win32 `SendInput`，发送 Unicode 字符，约 30 行 |
| 配置文件 | `serde` + `serde_json` | JSON 格式 |
| 日志 | `log` + `env_logger` | 标准方案 |
| 哈希去重 | `md-5` crate | MD5 哈希比对（与现有 PS 脚本一致） |

交叉编译方案：`cargo-xwin` + `x86_64-pc-windows-msvc` 目标（从 WSL2 构建原生 Windows EXE）

## 配置文件格式 (`config.json`，放在 EXE 同目录）

```json
{
  "hotkey": "Alt+Insert",
  "output_path": "/workspace/.clip/latest.png",
  "save_dir": ".clip",
  "poll_interval_ms": 800,
  "max_history_days": 7
}
```

- `hotkey`: 支持格式如 `"Alt+Insert"`, `"Ctrl+Shift+V"`, `"Super+V"` 等（由 global-hotkey 的 `HotKey::try_from()` 解析）
- `output_path`: 热键触发时输入到终端的**容器侧路径**（如 `/workspace/.clip/latest.png`）
- `save_dir`: 图片在 **Windows 侧**的保存目录。支持两种形式：
  - **相对路径**（如 `.clip`）：相对于 workspace 根目录，workspace 通过 EXE 所在位置向上推导（与 PS 脚本一致：`clipImg/clipimg-app/` → `clipImg/` → workspace root）
  - **绝对路径**（如 `E:\WorkingProjects\workspace\.clip`）：直接使用，不推导
- `poll_interval_ms`: 剪贴板轮询间隔
- `max_history_days`: 历史图片最大保留天数（clean 功能）

两个路径的关系：`save_dir` 是 Windows 文件系统上的实际写入位置，`output_path` 是 Docker 容器内能识别的路径，两者通过目录挂载映射到同一个物理文件。

## 核心逻辑

### 1. 剪贴板监控 (`clipboard.rs`)

移植现有 `clipboard-watcher.ps1` 的去重逻辑：
1. 轮询 `arboard::Clipboard::get_image()` 获取 RGBA 数据
2. 用 `image` crate 编码为 PNG，先写 `_tmp_clip.png`
3. 两级去重：文件大小 → MD5 哈希（与 PS 脚本完全一致）
4. 新图片：`_tmp_clip.png` → `clip_YYYYMMDD_HHmmss.png` + 覆盖 `latest.png`
5. 旧图片：删除 `_tmp_clip.png`

### 2. 热键 → 路径输入 (`input.rs`)

热键触发时：
1. 检查 `save_dir/latest.png` 是否存在
2. 存在：调用 Win32 `SendInput` 逐字符发送 `output_path`（Unicode 模式）
3. 不存在：弹出 tooltip 提示

### 3. 托盘菜单 (`main.rs`)

右键菜单项：
- **状态**（灰色文字）："运行中 / 监控目录: .clip"
- **打开配置文件**：用默认编辑器打开 `config.json`
- **打开图片目录**：用资源管理器打开 `save_dir/`
- **开机自启**（Checkbox）：读写注册表 `HKCU\...\Run`
- **退出**

### 4. 事件循环

```
main() → tao 事件循环
  ├── clipboard_timer (每 poll_interval_ms 触发) → clipboard.rs
  ├── GlobalHotKeyEvent::receiver() → input.rs
  ├── MenuEvent::receiver() → 托盘菜单处理
  └── TrayIconEvent::receiver() → 托盘点击
```

## 测试策略

### 单元测试（`#[cfg(test)]` 模块，跟源码放一起）

**`config.rs` 测试：**
- 默认配置生成正确
- 从 JSON 文件加载配置
- 不存在的配置文件 → 自动创建默认配置
- 无效 JSON → 报错不崩溃
- `save_dir` 相对路径 → 正确拼接为绝对路径
- `save_dir` 绝对路径 → 直接使用
- `hotkey` 字符串 → 解析为 HotKey 结构体
- 配置校验：hotkey 为空时拒绝

**`clipboard.rs` 测试：**
- MD5 去重：相同图片不重复保存
- MD5 去重：不同图片正确保存
- 文件大小不同 → 跳过 MD5 直接判定为新图片
- latest.png 不存在 → 任何图片都视为新图片
- `_tmp_clip.png` 残留 → 正确清理
- 时间戳文件命名格式 `clip_YYYYMMDD_HHmmss.png`
- 历史清理：超过 max_history_days 的文件被删除
- 历史清理后 latest.png 指向最新留存的图片

**`input.rs` 测试：**
- 无法在 Linux 上测试 SendInput（Win32 API），用 `#[cfg(target_os = "windows")]` 条件编译
- 测试字符到 INPUT 结构体的转换逻辑（不依赖实际 SendInput 调用）

### 集成测试（`tests/` 目录）

**`test_config.rs`：**
- 创建临时目录 → 写入 config.json → 加载 → 验证字段
- 修改 config.json → 重新加载 → 验证变更生效

**`test_clipboard.rs`：**
- 在临时目录模拟完整的剪贴板保存流程
- 创建测试 PNG 文件 → 写入 → 验证 latest.png 和 timestamped 文件
- 模拟去重场景

**`test_integration.rs`：**
- 端到端：配置加载 → 剪贴板保存 → 文件存在检查
- 标记 `#[ignore]`（需要 Windows 环境，`cargo test -- --ignored` 运行）

- 定义 `AppConfig` struct（Serialize/Deserialize）
- 加载 config.json，不存在则生成默认配置
- `save_dir` 路径解析（相对→绝对）
- 单元测试：默认值、加载、保存、路径解析、校验

### Step 3: `clipboard.rs` — 剪贴板图片保存 + 测试
- 实现 `ClipboardWatcher` struct
- `poll()` 方法：读剪贴板 → 去重 → 保存
- 文件操作：temp → timestamp → latest
- MD5 去重
- 历史清理逻辑
- 单元测试：去重、命名、清理、残留文件处理

### Step 4: `input.rs` — Unicode 文字输入

### Step 5: `main.rs` — 主程序集成

## 验证方案

1. **编译测试**：`cargo test`（Linux 上运行非 Windows 特定的测试）
2. **交叉编译测试**：`cargo xwin build --target x86_64-pc-windows-msvc --release` 生成 EXE
3. **Windows 功能测试**：
   - 复制图片 → save_dir 自动生成 PNG + `latest.png` 更新
   - 按配置的热键 → 终端出现 output_path 字符串
   - 右键托盘图标 → 菜单正常弹出
   - 修改 config.json 中 hotkey → 重启后新热键生效
   - 修改 output_path → 重启后输入新路径
   - 修改 save_dir 为绝对路径 → 图片保存到指定位置
4. **回归测试**：现有 bash 脚本 (`scripts/clipimg`) 不受影响

## 风险与应对

| 风险 | 应对 |
|------|------|
| `arboard` 返回的图片数据与 PS 脚本不一致 | 验证 RGBA→PNG 输出与 PS 的 `GetImage().Save()` 输出一致 |
| `cargo-xwin` 交叉编译链路出问题 | 备选：直接在 Windows PowerShell 里 `cargo build --release` |
| `global-hotkey` 与某些应用冲突 | 提供配置文件让用户换快捷键 |
| SendInput 在 UAC 提权窗口无效 | 这是 Windows 安全限制，无法绕过，在 README 注明 |
