# clipImg v1.0.1 改进计划

## 改进项

### 1. 首次运行确认 save_dir 路径

**现状**：程序根据 EXE 位置自动推导 save_dir（向上两级），路径可能不符合用户预期，且没有确认环节直接创建。

**改进**：
- 首次运行时弹出可编辑输入框，展示推导出的实际路径
- 用户可修改路径后点确认，再创建目录和配置文件
- 取消则退出程序

### 2. 历史保留时间改为小时级别

**现状**：`max_history_days: 7`，粒度为天。

**改进**：
- 新字段 `max_history_hours`，默认值 `1`（保留 1 小时）
- 不兼容旧字段 `max_history_days`，忽略旧值，使用新字段默认值
- `latest.png` 始终保留，不受过期清理影响
- `clip_*.png` 超过 max_history_hours 后删除

### 3. 去掉启动时的控制台黑框

**现状**：双击 EXE 弹出黑色控制台窗口。

**改进**：
- Windows 子系统改为 `WINDOWS`（`#![windows_subsystem = "windows"]`），双击运行无黑框
- 托盘菜单增加「打开日志」选项，用记事本打开 `.clipimg.log`
- 支持 `--console` 命令行参数，带此参数启动时附加控制台窗口

### 4. 应用图标

**现状**：EXE 无图标，任务栏托盘白底空图标。

**改进**：
- 生成一个简约图标（剪贴板 + 图片的符号组合）
- 通过 `.rc` 资源文件嵌入 EXE
- 同时用作系统托盘图标

### 5. 开机自启开关

**现状**：无开机自启功能。

**改进**：
- 托盘菜单增加「开机自启」勾选项，即时生效
- 写入/删除注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
- 启动时根据注册表状态初始化勾选状态

---

## 实施顺序

| 步骤 | 改进项 | 涉及文件 |
|------|--------|----------|
| 1 | 去掉黑框 + --console | `main.rs`, `Cargo.toml` |
| 2 | 应用图标 | 新增 `resource.rc`, `Cargo.toml` |
| 3 | 开机自启 | `main.rs` |
| 4 | 历史保留改小时 | `config.rs`, `clipboard.rs` |
| 5 | 首次运行确认路径 | `main.rs`, `config.rs` |
