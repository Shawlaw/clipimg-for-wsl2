# v1.0.11 计划

## 变更内容

### 新增：托盘菜单"打开程序目录"

**需求**：用户下载新版 clipImg 后需要找到当前 EXE 位置覆盖替换，目前没有快捷入口。

**改动**：

1. **main.rs — 新增菜单项**
   - 在"打开图片目录"下方新增 `MenuItem::with_id("open_exe_dir", "打开程序目录", true, None)`
   - 菜单顺序：`open_log → open_config → reload_config → open_dir → open_exe_dir → homepage`

2. **main.rs — 点击处理**
   - `"open_exe_dir"` 分支：用 `std::current_exe()` 获取 EXE 路径，调用 `desktop_fs::open_path()` 打开所在目录
   - 复用现有的 `desktop_fs::open_path` 函数，无需额外依赖

3. **版本号**
   - `Cargo.toml` version 更新为 `1.0.11`
   - `CHANGELOG.md` 新增 v1.0.11 条目

## 不做的事

- 不新增"打开配置目录"（已有"打开配置文件"直接打开 config.json，功能重叠）
- 不做自动更新机制（超出本次范围）

## 验证

- 编译通过，托盘菜单出现"打开程序目录"项
- 点击后资源管理器打开 EXE 所在目录
- EXE 在含空格/中文的路径下正常工作
