# v1.0.12 计划

## 变更内容

### 1. Debug 构建模式（调测便利性）

**现状**：每次测试新版本需要手动退出当前运行的 exe → 改名 → 复制新 exe → 运行，流程繁琐。

**方案**：新增 `debug_build` feature，编译出 `clipimg_debug.exe`。启动时自动终止 release 版本进程，实现"复制过去双击即测"。

**改动**：

#### Cargo.toml
- 新增 `debug_build` feature
- 新增 `[[bin]]` target：`name = "clipimg"` 和 `name = "clipimg_debug"`（后者 `required-features = ["debug_build"]`）

#### main.rs — 启动时终止 release 进程
- `#[cfg(feature = "debug_build")]` 块中调用 `taskkill /IM clipimg.exe /F`
- 使用 `CREATE_NO_WINDOW` 标志避免黑框闪过

#### main.rs — 互斥体名称
- debug 版本使用 `Global\clipimg_debug`，避免与 release 版冲突
- 通过 `cfg!` 宏选择 mutex 名称

#### main.rs — 托盘 tooltip 标记
- debug 版本在 tooltip 和状态菜单中显示 `(debug)` 标记，方便区分

#### main.rs — 开机自启状态
- debug 版本 `is_autostart_enabled()` 只检查注册表是否存在 `clipImg` 值，不比较 exe 路径
- 因为 debug 版本操作的是 release 版本的自启项

#### 构建命令
- 本地调试：`cargo xwin build --target x86_64-pc-windows-msvc --features debug_build --release --bin clipimg_debug`
- Release 构建不变：`cargo xwin build --target x86_64-pc-windows-msvc --release`
- GitHub Actions 无需修改，只编译 release 版本
